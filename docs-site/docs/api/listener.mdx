# Listener

**Listener** — точка приёма трафика балансировщика. Он отвечает на вопрос «на каком порту и
протоколе принимать соединения и на какой порт таргета их пересылать». Бизнес-смысл: один
[NetworkLoadBalancer](/api/network-load-balancer) с одним VIP может обслуживать несколько
портов/протоколов — каждый описывается отдельным листенером (например, `TCP:80` и `TCP:443`).

Listener — **самостоятельный ресурс** (собственный ID `lst`, свой FGA-объект и своя история
операций), но живёт под конкретным балансировщиком (`loadBalancerId`, FK) и **наследует его
VIP** — собственного адреса листенер не несёт. Это осознанное решение Kachō: один anycast-VIP
на балансировщик на семейство, а листенеры мультиплексируются по портам.

:::note Статус независим от балансировщика
`Listener.status` отделён от `NetworkLoadBalancer.status`: листенер может быть `UPDATING`, пока
родительский балансировщик остаётся `ACTIVE`.
:::

## Методы

Чтение (`Get` / `List` / `ListOperations`) — синхронное. Все мутации асинхронны и возвращают
[`Operation`](/api/operations).

### Get — получить листенер

<ApiOperation method="GET" endpoint="/nlb/v1/listeners/{id}">

Возвращает листенер по идентификатору.

<CodeBlock language="json">
  {dedent`
    {
      "id": "lst...",
      "loadBalancerId": "nlb...",
      "projectId": "prj-1",
      "name": "http",
      "protocol": "TCP",
      "port": 80,
      "targetPort": 80,
      "defaultTargetGroupId": "tgr...",
      "status": "ACTIVE",
      "createdAt": "2026-06-24T10:00:00Z"
    }
  `}
</CodeBlock>

</ApiOperation>

### List — список листенеров

<ApiOperation method="GET" endpoint="/nlb/v1/listeners">

Возвращает листенеры проекта (`projectId` обязателен). Опциональный `loadBalancerId`
ограничивает выборку одним балансировщиком. Результат фильтруется по доступу (viewer ∪
`v_list`). `filter` поддерживает `name = "..."`.

<CodeBlock language="bash">
  {dedent`
    curl 'http://localhost:18080/nlb/v1/listeners?projectId=prj-1&loadBalancerId=nlb...' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

</ApiOperation>

### Create — создать листенер

<ApiOperation method="POST" endpoint="/nlb/v1/listeners" async>

Создаёт листенер под указанным балансировщиком. `loadBalancerId`, `protocol`, `port`,
`targetPort` обязательны; порты — в диапазоне 1–65535 (0 недопустим). Листенер наследует VIP
балансировщика.

<CodeBlock language="bash">
  {dedent`
    curl -X POST 'http://localhost:18080/nlb/v1/listeners' \\
      -H 'Authorization: Bearer <JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{
            "loadBalancerId": "nlb...",
            "name": "https",
            "protocol": "TCP",
            "port": 443,
            "targetPort": 8443,
            "proxyProtocolV2": true,
            "defaultTargetGroupId": "tgr..."
          }'
  `}
</CodeBlock>

</ApiOperation>

### Update — изменить листенер

<ApiOperation method="PATCH" endpoint="/nlb/v1/listeners/{id}" async>

Меняет изменяемые поля (`name`, `description`, `labels`, `targetPort`, `proxyProtocolV2`,
`defaultTargetGroupId`). Immutable-поля (`loadBalancerId`, `protocol`, `port`) в `updateMask`
отклоняются (`InvalidArgument`).

<CodeBlock language="bash">
  {dedent`
    curl -X PATCH 'http://localhost:18080/nlb/v1/listeners/lst...' \\
      -H 'Authorization: Bearer <JWT>' \\
      -H 'Content-Type: application/json' \\
      -d '{ "updateMask": "targetPort", "targetPort": 9000 }'
  `}
</CodeBlock>

</ApiOperation>

### Delete — удалить листенер

<ApiOperation method="DELETE" endpoint="/nlb/v1/listeners/{id}" async>

Удаляет листенер.

<CodeBlock language="bash">
  {dedent`
    curl -X DELETE 'http://localhost:18080/nlb/v1/listeners/lst...' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

</ApiOperation>

### ListOperations — история операций

<ApiOperation method="GET" endpoint="/nlb/v1/listeners/{id}/operations">

Возвращает историю асинхронных операций листенера (cursor-пагинация). Формат элемента —
[`Operation`](/api/operations).

</ApiOperation>
